    二分查找总结

    1. 基础二分

    在闭区间[l,r]查找给定数target，终止条件l>r，若找不到则返回-1

    int bs1(int nums[], int target){
    int l = 0, r = nums.length - 1; //左闭右闭区间
    while(l<=r){ //终止条件l>r
        int mid = (l+r)/2;
        if(nums[mid] == target) return mid;
        else if(nums[mid] > target) r = mid-1;
        else if(nums[mid] < target) l = mid+1;
    }
    return -1; //循环终止了仍找不到就是找不到，此时搜索区间已空
}

    2. 确定下界的二分

    在闭区间[l,r]查找给定数target，终止条件l>r

    • 若target在nums中有多个，则返回最小的下标
    • 若target不存在于nums中，返回大于target的最小的数的下标
    • 若target不存在于nums中且大于nums中最大的数，返回nums.length
    int lower_bound(int nums[], int target) {
    int l = 0, r = nums.length - 1;
    while (l <= r) {
        int mid = (l + r) / 2;
        if (nums[mid] == target)
            r = mid - 1 ; //即使找到了也是继续缩小右边界而非返回
        else if (nums[mid] > target)
            r = mid - 1;
        else if (nums[mid] < target)
            l = mid + 1;
    }
    return l;
}

    3. 确定上界的二分

    在闭区间[l,r]查找给定数target，终止条件l>r

    • 若target在nums中有多个，则返回最大的下标
    • 若target不存在于nums中，返回不超过target的最大的数的最大下标
    • 若target不存在于nums中且小于nums中最小的数，返回-1
    int upper_bound(int nums[], int target){
    int l = 0, r = nums.length - 1;
    while(l <= r){
        int mid  = (l+r)/2;
        if(nums[mid] == target)
            l = mid + 1;
        else if(nums[mid] > target)
            r = mid - 1;
        else if(nums[mid] < target)
            l = mid + 1;
    }
    return r;
}

    4. 二分答案

    最基础的模型就是猜数字游戏，每次以二分的方法给出一个答案，并判断答案是大了、小了还是刚好，直到判断结果为刚好/搜索区间为空为止

    二分过程套模板，但judge函数需要根据题目条件编写，但最终目的都是一样的，就是用来judge二分过程给出的答案是大了、小了还是刚好。

    • 1011.在d天内送达包裹的能力

      class Solution {
public:
    //判断二分出的运载能力x是否能使得船在days天内运输完货物
    int judge(vector<int>& weights, int x, int days){
        int cnt = 0;
        int now = 0;
        for(int i = 0; i< weights.size();i ++){
            if(now == 0 && now + weights[i] > x) return -1;
            if(now + weights[i] > x){
                cnt ++;
                now = 0;
                i--;
            }
            else now += weights[i];
        }
        cnt ++;
        //cnt统计所需的天数
        if(cnt > days) return -1; //运载能力小了，要更大
        else if(cnt == days) return 0; //运载能力刚刚好
        else return 1; // 运载能力大了，可以再小点
    }
    int shipWithinDays(vector<int>& weights, int days) {
        int sum = 0;
        int max = 0;
        for(int i = 0;i <weights.size();i++){
            sum += weights[i];
            if(weights[i] > max) max = weights[i];
        }
        int l = max, r = sum;
        while(l <= r){
            int mid = l + (r-l)/2;
            int res = judge(weights, mid, days);
            if(res == 1){
                r = mid - 1;
            }
            else if(res == -1){
                l = mid + 1;
            }
            else{
                r = mid - 1;
            }
        }
        return l;
    }
};

    • 410.分割数组的最大值

      该题需要确定的是二分枚举的应该是个什么东西。

      要求将数组划分为m个子数组，显然有很多种不同的划分方式，而这是无法枚举的

      再看题目给出的关键词：最大值最小。

      “最小”这个条件可以联想到确定下界的二分。如果将这个“最大值”作为二分枚举的对象，如何judge它是大了、小了还是刚好呢？

      若最大值max已经确定，则数组的划分方式也可以同时确定了：贪心。每次都将尽可能多的元素划分为一组，直到子数组和超出了这个最大值，再开新的一组，若最终得到的总组数为m，max就是“刚好”，若小于m，max就是“大了”，max可以再小一些，使得子数组组数更大一些；若大于m，max就是“小了”。

      由于子数组中至少要有1个元素，因此二分枚举的l起始值应该为数组中的最大值，确保划分数组的过程中不会出现空的子数组。

      通过以上分析，可以知道这道题本质上和1011是一样的，连judge函数都是一样的。

      class Solution {
public:
    //这里二分要枚举的是子数组和的最大值
    //再看该最大值限制下贪心划分是否能恰好划分出题目要求的k个子数组
    int judge(vector<int>& nums, int max, int k){
        int cnt = 0;
        int now = 0;
        for(int i = 0; i < nums.size(); i ++){
            if(now + nums[i] > max){
                now = 0;
                cnt ++;
                i --;
            }
            else now += nums[i];
        }
        cnt ++;
        if(cnt > k) return -1; //小了
        else if(cnt < k ) return 1; //大了
        else return 0;//刚好

    }
    int splitArray(vector<int>& nums, int k) {
        int sum =0, max = 0;
        for(int i = 0; i < nums.size();i++){
            sum+=nums[i];
            if(nums[i] > max) max = nums[i];
        }
        int l = max, r = sum;
        while(l<=r){
            int mid = (l + r)/2;
            int res = judge(nums, mid, k);
            if(res == -1){
                l = mid+1;
            }
            else if(res == 1){
                r = mid -1;
            }
            else r = mid -1;
        }
        return l;
    }
};

    • 2528.最大化城市的最小供电站数目

      题目表述略有问题，”电量“这个词除了给出定义以外就再也没出现过了，但结合上下文来看，最后问的其实是所有城市中最小电量的最大值；示例1的解释里- 城市 0 的供电站数目为 1 + 4 = 5 。其实应该是- 城市 0 的电量为 1 + 4 = 5 。

      • 第一步，确定二分法

        关键词最小供电站数目（应该是电量）的最大值是多少，”最小值的最大值“这种表述十有八九是二分，且二分要枚举的对象就是这个最小电量.

        然后确定起始的left和right应该是多少：

        • 考虑最小电量的下界，建额外供电站只会增大一个城市的电量，因此初始情况下所有城市电量的最小值就是left，即left = min(power[i])；
        • 虑最小电量的上界，最极端的情况就是把所有的$k$个额外供电站都堆在初始情况下电量最大的城市，其他任何建法所能达到的电量都不会超过这个值，即right = max(power[i]) + k

        分析到这里，二分模板可以先打一个：

        //power[i]即初始时城市i的电量（不是供电站数目）
int left = min(power[i]), right = max(power[i]) + k;
while (left <= right){
    int mid = left + (right - left)/2; //枚举最小电量
    if(judge(...)){ 
        //如果judge函数返回True，则mid这个最小电量是可以通过建额外供电站达到的，可以把这个最小电量再增大一点试试
        left = mid + 1;
    }
    else{
        //如果judge函数返回False，则mid这个最小电量太大了，怎么建供电站都不可能满足，需要缩小一点
        right = mid - 1;
    }
}
return right; //求的是最大值，即上界

      • 第二步，思考如何判定二分枚举的最小电量是否符合要求（如何写judge函数）

        有了最小电量mid，接下来是考虑怎么建供电站使得所有的power[i]都≥mid

        显然这个建法有很多种，注意，如果开始纠结”假如这样安排也可以，那样安排也可以“，通常答案是贪心地安排

        （属于是二分题的一种套路吧，类似的题目有1011. 在 D 天内送达包裹的能力，二分+贪心地把尽可能多的货物安排在同一天运；以及410. 分割数组的最大值，二分+贪心地将尽可能多的元素纳入同一个子数组）

        假设范围r=2，我们需要将power[3]增大1，则贪心建法是让城市3位于新建供电站的最左侧，即让power[3]、power[4]、power[5]、power[6]、power[7]全部增大1。

        看得出来我们需要频繁地进行【令power[l...r]全部加x】这样的操作，很自然地想到差分数组（直接简单粗暴每次都遍历加一遍的话会超时）

        分析到这里，可以写出judge函数的伪代码：

        bool judge(vector<long long> diff, vector<long long> power, long long minPower, int k){ 
    for(int i = 0; i < power.size(); i++){
        if(i > 0) power[i] = power[i-1] + diff[i];
        //城市i前面可能新建了供电站，需要先更新城市i的电量
        if(power[i] < minPower){ 
            long long x = minPower - power[i];
            if(k > x) return false; //已经不够建了，minPower太大，无法达到
            else{
                //令power[i...i+2*r]全部加x
                diff[i] += x;
                if(i + 2 * r + 1 < power.size()){
                    diff[i + 2 * r + 1] -= x;
                }
                k -= x;
            }
        }
        //更新城市i的电量
        if (i == 0)
            power[i] = diff[i];
        else
            power[i] = power[i - 1] + diff[i];
    }
    return true; //可以通过贪心地建供电站使得所有城市电量都≥minPower
}

        到这里其实已经基本写完了，我们已经写出了【二分枚举minPower】+【判定这个minPower是大了还是小了】两个核心部分。

      • 还剩最后一个问题：power数组怎么求？

        由题意知，城市i的电量power[i]其实就是stations[i-r...i+r]的区间和，快速求区间和，那就用到前缀和。

        long long sum[stations.size() + 1];
sum[0] = 0;
for(int i = 1; i <= stations.size(); i++){
    sum[i] = stations[i - 1] + sum[i - 1];  //计算前缀和数组
}
for(int i = 0;i < stations.size(); i++){
    int begin = 0 > i - r ? 0 : i - r;
    int end = stations.size() - 1 < i + r ? stations.size() - 1 : i + r;
    // 计算stations[i-r ... i+r]的区间和作为power[i]，注意左右不能越界
    power.push_back(sum[end + 1] - sum[begin]);
}

    5. 旋转数组问题

    思路

    题目特点是所给数组为一个升序数组，分为两段调换前后位置的结果，特点是两段升序数组，且有nums[0]>nums[n-1]

    • 分段处理法
      • 首先二分找到第二段的起始下标k
      • 各段各自进行二分搜索

    题目

    • 寻找旋转排序数组中的最小值（不含重复数字）

      只需要找到分段点nums[k]即可

      由于nums[0]>nums[n-1]，故

      • 当nums[mid]<nums[r]时，mid一定位于第二段，k可能在mid左侧，也可能就是mid，因此需要r = mid
      • 当nums[mid]>nums[r]时，mid一定位于第一段，k一定在mid右侧，因此需要l = mid + 1

      • 当l == r时，l=r=mid即所求位置，为了让循环能终止，应执行l = mid + 1后返回r，故还需要加上nums[mid]==nums[r]时l = mid + 1，即：

        int l = 0, r = nums.size() - 1;
while(l <= r){
    int mid = l + (r - l)/2;
    if(nums[mid] < nums[r]) r = mid;
    else l = mid + 1;
}
      • 最后所得r即为所求k

    • 搜索旋转排序数组（不含重复数字）

      • 找到第二段的起始下标k
      • 分为前后两段各自进行二分查找
      class Solution {
public:
    int find(vector<int>& nums, int l, int r, int x){
        while(l<=r){
            int mid = l + (r-l)/2;
            if(nums[mid] == x) return mid;
            else if(nums[mid] < x) l = mid +1;
            else r = mid - 1;
        }
        return -1;
    }
    int search(vector<int>& nums, int target) {
        int l = 0, r = nums.size()-1;
        while(l<=r){
            int mid = l + (r-l)/2;
            if(nums[mid] >= nums[r]) l = mid + 1;
            else r = mid;
        }
        int pos = find(nums, 0, r-1, target);
        if(pos == -1) pos = find(nums, r, nums.size()-1, target);
        return pos;
    }
};

    • 寻找旋转排序数组中的最小值 II（含重复数字）

      由于含重复数字，我们并不能默认nums[mid]==nums[r]时mid一定就是最小值的下标。

      当nums[mid] == nums[r]时：

      • 情况一：mid和r和k位于第二段的同一处平台，nums[r]恰是最小值
      • 情况二：mid和r位于第二段的同一处平台，k位于该平台左侧

      此时令r--：

      • 若为情况一，则nums[r]可能仍是最小值，会继续减1直到跳出循环；或nums[r]是最大值，则接下来必然跳出循环，nums[l]为答案
      • 若为情况二，r可以通过r = mid -1直接移到平台外，或通过r--移到平台外，最终转化为情况1

      综上，需要特别注意的是r--这个操作，以及最终返回的答案应该是nums[l]

      class Solution {
public:
    int findMin(vector<int>& nums) {
        int l = 0, r = nums.size() - 1;
        while(l <= r){
            int mid = l + (r - l)/2;
            if(nums[mid] < nums[r]){
                r = mid;
            }
            else if(nums[mid] > nums[r]) {
                l = mid + 1;
            }
            else if(nums[mid] == nums[r]){
               //mid和r都位于第二段的同一段平台，k要么在同一平台，要么在平台左边
                r--;
            }
        }
        return nums[l];
    }
};
